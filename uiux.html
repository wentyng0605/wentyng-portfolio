UI/UX Case Study

Bloom-In-Motion: Ticketing Web App

Role: UI/UX Designer

The Challenge

Managing event registration for university clubs can be chaotic. Traditional manual sign-ups often lead to fragmented information and high cognitive load for students. The goal was to design a mobile-friendly ticketing web app that simplifies seat selection and payment verification.

The Process

Requirement Gathering: User interviews with club representatives identified "information clarity", "registration traffic" and "seat availability" as top priorities.

Flow Optimization: Applied HCI principles to design an intuitive user flow that minimizes booking steps.

Design Strategy & HCI Implementation

Conversion-Optimized Landing

High-contrast hero section with clear CTA and clean information hierarchy for event specifics.

Interactive Seat Selection

Applied Fitts's Law to optimize button placement. Color-coded auditorium layout minimizes cognitive load.

Frictionless Payment

Progressive Disclosure breaks down forms into logical steps, preventing "form fatigue" and reducing drop-off.

Instant System Feedback

Visually distinct confirmation screen provides immediate closure following Nielsen's Visibility of System Status.

UI/UX Case Study

TravelGo: Redesign Travel Apps

Role: UX Researcher & UI Designer (HCI Academic Project)

The Challenge

Travelers often struggle to balance itinerary logistics with budget constraints. My goal for this HCI project was to design a seamless management solution that integrates hotel/flight booking with real-time budget tracking, validated through rigorous user testing.

User Research & Insights

Conducted quantitative research with 25 participants to measure usability and user satisfaction.

User Demographic: Target audience aged 18-25, focusing on tech-savvy frequent travelers.

Testing Setup: Within-Subject Design where users performed tasks (Login, Explore Navigation, Feedback) to test intuitiveness.

Design Strategy & Heuristic Evaluation

Leveraging Nielsen's 10 Usability Heuristics, I ensured the interface feels familiar and reliable.

Recognition Over Recall

Clear iconography and visual cues reduce cognitive load. Users recognize actions instead of remembering locations.

System Match to Real World

Familiar gestures (pinch-to-zoom, drag) and logical booking flow mirror real-world sequences.

Consistency & Standards

Strict visual language across the app ensures users feel confident navigating.

The Results & Discussion

The prototype was evaluated using the System Usability Scale (SUS) for quantitative validation.
